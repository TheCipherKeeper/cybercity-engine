# ADR-0006: Go-first implementation

## Status

Accepted

## Context

Первоначальный план предполагал Python reference-реализацию с последующим
Go-портом для production. После обсуждения пришли к выводу, что CyberCity Engine
с самого начала выиграет от системного языка:

- Движок — это runtime с высоким параллелизмом (события, WebSocket, heartbeat,
  background-процессы).
- Два графа (топологический и событийный) и event routing удобно выражать
  структурами и goroutines.
- Производительность, компилируемость и статическая типизация Go снижают риски
  на этапе home lab и публичной демонстрации.

## Decision

Реализовать **Go-first reference-версию** движка, заменив ей Python-код.

Python-версия не сохраняется как reference: её функциональность переносится
идентичной семантикой в Go, а тесты и ADR фиксируют поведение.

## Обоснование

- **Concurrency.** Goroutines + channels естественно моделируют tick-цикл,
  очередь команд и параллельную обработку событий без GIL.
- **Производительность.** Меньшее потребление памяти и latency на hot path.
- **Статическая типизация.** Ошибки моделей и router-правил ловятся на этапе
  компиляции.
- **Развёртывание.** Один бинарник + Docker-образ без интерпретатора и
  зависимостей Python-экосистемы.
- **Графы.** Топологический и событийный графы эффективно хранятся в
  map/struct-структурах; обходы — простые циклы.

## Границы

- Go-движок должен сохранить ту же архитектуру: два графа, событийный runtime,
  engine как единственный мутатор состояния.
- Модели остаются чистыми данными.
- Handlers и router rules — чистые функции.
- Persistence и messaging пока остаются интерфейсами/заглушками, как было в
  Python-скелете.

## Consequences

### Positive

- Более высокая производительность из коробки.
- Проще горизонтальное масштабирование и бинарное развёртывание.
- Явная типизация моделей и событий.
- Go-привычки (go test, go vet) подходят для CI/CD.

### Negative

- Порог входа для контрибьюторов без Go-опыта выше, чем у Python.
- Нужно переписать тесты и документацию.
- Нет ready-to-use экосистемы Pydantic/FastAPI; придётся писать валидацию
  и сериализацию явно.
